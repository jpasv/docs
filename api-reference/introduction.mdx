## Autenticação

A API do BullQ usa autenticação via **Bearer Token**. Obtenha seu token fazendo login:

<CodeGroup>
```bash cURL
curl -X POST "https://api.bullq.com.br/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email": "seu@email.com", "password": "sua_senha"}'
```

```javascript JavaScript
const response = await fetch('https://api.bullq.com.br/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'seu@email.com', password: 'sua_senha' })
});
const { accessToken } = await response.json();
```
</CodeGroup>

Use o token em todas as requisições:

```bash
Authorization: Bearer seu_token_aqui
```

## Rate Limits

| Tier | Limite | Janela |
|------|--------|--------|
| Short | 10 req | 1s |
| Medium | 100 req | 1min |
| Long | 1000 req | 15min |

## Códigos de Status

| Código | Descrição |
|--------|-----------|
| 200 | Sucesso |
| 201 | Criado com sucesso |
| 400 | Requisição inválida |
| 401 | Não autorizado |
| 403 | Acesso negado |
| 404 | Não encontrado |
| 429 | Rate limit excedido |
| 500 | Erro interno |
